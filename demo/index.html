2. Step-by-Step Cryptographic Flow

Equation: \(G_x = \text{Encode}(x) = \bigcup_{i=0}^{k-1} (x_i \cdot \mathbf{1}^r)\) where \(k=8, r=8, N=64\)
To protect the plaintext against the chaotic mixing of cellular automata diffusion and enable homomorphic evaluation, the 8-bit plaintexts \(A\) and \(B\) are expanded into 64-bit arrays. Each bit of the number is repeated 8 times (repetition factor \(r=8\)). This redundancy allows the majority-vote decoder to correct any diffusion noise.

An Initial Vector (IV) is generated from the seed. The encoded grid state is masked using a bitwise XOR with the IV. This state constitutes the starting grid at time \(T=000\). The second-order reversible CA requires two historical steps to run: \(S_0\) (the masked plaintext grid) and \(S_{-1}\) (the IV itself).

Second-Order CA Evolve: \(S_{t+1} = f(S_t) \oplus S_{t-1}\)
The initial state grid undergoes \(T=50\) steps of evolution using a reversible second-order rule (Fredkin's involution). The next cell state is determined by a local function \(f\) (defined by the ruleset and neighborhood radius) of the current cell and its neighbors, XORed with the historical cell state. Because the CA is second-order, it is completely bijective, ensuring zero loss of entropy during encryption.

Decryption Reverse: \(S_{t-1} = f(S_t) \oplus S_{t+1}\)  |  Voting: \(x'_i = \text{Majority}(G'_i)\)
To decrypt, the client reverses the evolution of the evaluated grid by running the CA backwards for \(T=50\) steps. The decrypted grid is then XORed with the combined IV to retrieve the combined plaintext grid. Finally, a majority vote is performed on each block of 8 bits to decode the final result. Even if chaotic diffusion introduced bit flips, the majority-voting filter isolates and corrects the errors.
